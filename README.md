# Unsupervised clustering of high-dimensional flow cytometry data

A compact, reproducible pipeline for **unsupervised analysis of multicolor flow
cytometry data** — the core computational task in immune monitoring and
translational immuno-oncology. It demonstrates the standard FlowSOM-style
workflow used across the field (Weber & Robinson, *Cytometry A* 2016; Nowicka
et al., *F1000Research* 2019) end to end, in Python.

The project does two things:

1. **Validates the method on simulated data with a known ground truth**, so
   clustering accuracy can be *measured* (Adjusted Rand Index = **0.996**).
2. **Applies the identical pipeline to a real 16-color `.fcs` file**, where
   clusters are interpreted from their marker phenotypes — the everyday task of
   an immune-monitoring analyst.

---

## The pipeline

Both analyses share one module (`src/flow_pipeline.py`) and the same five steps:

| Step | What | Why |
|------|------|-----|
| 1. Load | read `.fcs` with FlowKit, apply the embedded compensation matrix | corrects fluorophore spillover between detectors |
| 2. Transform | `arcsinh` (cofactor 150) | compresses the ~0–260,000 dynamic range; linear near zero, log for bright signal |
| 3. Scale | z-score each marker | stops wide-range markers from dominating the distance metric |
| 4. Cluster | **FlowSOM-style**: 10×10 Self-Organizing Map → metacluster the 100 nodes | fast, reproducible, benchmark-leading for cytometry |
| 5. Embed | UMAP (for *visualization only*) | 2-D view of structure; clustering is always done in full marker space |

Clusters are then annotated from a **median-marker-expression heatmap**.

## Results

### 1. Validation on simulated PBMC (ground truth known)
The pipeline recovers all seven simulated populations almost perfectly.

![simulated UMAP](results/figures/sim_umap_truth_vs_cluster.png)

Each cluster's phenotype maps cleanly to a known cell type
(B cell = CD19⁺HLA-DR⁺, NK = CD56⁺, CD4/CD8 naive vs memory by CD45RO/CCR7, etc.):

![simulated heatmap](results/figures/sim_marker_heatmap.png)

### 2. Real 16-color T-cell panel (`100715.fcs`)
The same pipeline, run on real data after light quality-control gating.
Marker expression projected onto the UMAP shows the underlying biology; the
heatmap (`results/figures/real_marker_heatmap.png`) is used to annotate clusters.

![real markers](results/figures/real_umap_markers.png)

Real data is messier than simulation — populations form gradients rather than
crisply separated islands, and final annotation requires domain expertise. That
contrast is the point: simulation proves the method is correct; real data shows
how it behaves in practice.

## Run it yourself

```bash
pip install -r requirements.txt

cd src
python 01_make_simulated_data.py     # writes data/simulated_pbmc.parquet
python 02_analyze_simulated.py       # validation + ARI + figures
python 03_analyze_real_fcs.py        # real .fcs analysis + figures
```

Outputs land in `results/figures/` and `results/tables/`.

## Notes & honest limitations
- `simulated_pbmc.parquet` is **synthetic** (see `data/DATA_SOURCES.md`); it
  exists to provide a ground truth for accuracy measurement, not to mimic any
  specific patient sample.
- The real file is a single sample distributed for software testing; it is used
  to demonstrate real `.fcs` I/O and the clustering workflow, not to draw
  biological conclusions.
- A natural extension is **differential abundance** across a multi-sample cohort
  (e.g. baseline vs on-treatment), which in R is handled by `diffcyt`/`CATALYST`.
